        • void forciblyReleaseMutex(java.lang.String identifier)

          Releases a mutex without requiring the key. WARNING: THIS IS DANGEROUS This method should only be used as a last resort to restore stability in an application. The application logic should be robust enough so that processes that acquire a mutex have the proper error handling to release it to avoid weird application states where the mutex is no longer respected.
